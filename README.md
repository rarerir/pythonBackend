# Сервис автоматизации статуса кампаний

REST API на FastAPI + SQLAlchemy + PostgreSQL с движком правил для управления рекламными кампаниями.

## Запуск проекта

**Требования:** Docker, Docker Compose v2.

1. Клонируйте репозиторий и перейдите в папку проекта.
2. Создайте файл `.env` на основе `.env.example` (можно оставить как есть).
3. Выполните:
   ```bash
   docker compose up -d --build
API будет доступен по адресу http://localhost:8000.
Dev UI: http://localhost:8000/docs.

При первом запуске автоматически создаются таблицы, типы, триггеры и тестовые данные.

Запуск тестов
Unit-тесты (проверяют правила, не требуют сервера):

bash
pytest tests.py -v -m "not api"
API-тесты (требуют работающий сервер на localhost:8000):

bash
pytest tests.py -v -m api
Если сервер запущен на другом порту:

bash
API_BASE_URL=http://127.0.0.1:9000 pytest tests.py -v -m api

Архитектура движка правил
Целевой статус кампании вычисляется последовательным применением правил.
Первое сработавшее правило определяет результат; если ни одно не сработало – статус active.

Каждое правило – отдельный класс с единым интерфейсом:
class Rule(ABC):
    @abstractmethod
    def evaluate(campaign, current_datetime) -> Optional[Tuple[CampaignStatusEnum, str]]:
        ...
Правила добавляются в список RULES_LIST и сортируются по приоритету. Движок обходит их по порядку и останавливается на первом, вернувшем результат.

Реализованные правила (по приоритету):
IsManaged – если управление выключено, статус не меняется.
ScheduleEnabled – если расписание активно и текущее время вне всех окон → paused.
LowStockRule – если осталось меньше дней, чем минимальный порог → paused.
BudgetExceededRule – если дневной расход ≥ лимита → paused.

При вызове POST /campaigns/{id}/evaluate создаётся запись в rule_evaluation_log со снапшотом данных кампании, сработавшим правилом и изменением статуса.
